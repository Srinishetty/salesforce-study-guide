2. Scenario: We have an "Advisers" object. When an Adviser creates a record, it should be editable by them, but NOT visible to other Advisers. However, a Sales Manager should be able to see and edit ALL Adviser records. How do you design this?

🎤 How to answer naturally
You: "Okay, so this is actually a pretty straightforward standard security model scenario. We wouldn't need any code for this." "First, I'd go into Sharing Settings and set the OWD (Org-Wide Default) for the Advisers object to 'Private'. That instantly locks it down so Advisers can only see their own records. They won't see their peers' data." "Then, to handle the Sales Manager part, I'd rely on the Role Hierarchy. I'd make sure we have an 'Adviser' role, and the 'Sales Manager' role sits right above it. As long as 'Grant Access Using Hierarchies' is checked, the Manager automatically gets full view and edit access to everything their team owns."
👤
Tech Panel

3. When a Lead is converted, automatically create a Task for the owner of the account. How will you achieve this? What are the possibilities?

🎤 How to answer naturally
You: "We could do this with Apex or a Flow, but honestly, these days I'd definitely go with a Record-Triggered Flow. It's much easier to maintain." "I'd set up an After-Save Flow on the Lead object. The trigger condition would just check if `IsConverted` equals true. Then I'd drag in a 'Create Records' element to make the Task, setting the `WhatId` to the newly converted Account's ID, and the `OwnerId` to the Account Owner." "If the client really insisted on Apex, I'd just write an `after update` trigger, loop through `Trigger.new` to check for that conversion flag, query the account owners, and do a bulk insert of the tasks. But Flow is the way to go here."

4. An Account has multiple Contacts. A user shouldn't be able to delete a Contact if it is the "last associated" one. How do you solve this?

🎤 How to answer naturally
You: "This is an interesting one. I'd actually ask the business for a quick clarification first. Does 'last associated' mean we just can't leave the Account empty, or does it mean the chronologically newest contact?" "If they just mean 'don't leave the account empty', we could use a Before-Delete Flow with a Custom Error. But to be fully safe against users doing mass-deletes, I prefer an Apex trigger. I'd do an aggregate query to count the total contacts, subtract the amount they are trying to delete, and if that math drops below 1, I use `.addError()` to block it." "If they actually mean 'don't delete the newest contact', I can't trust the standard `CreatedDate` because a contact could be created on Monday but linked to the account on Friday. So I'd use a Flow to stamp a custom 'Linked Date' field, and then write an Apex trigger to query the contacts ordered by that custom date to protect the newest one."

13. Write a trigger on account.

🎤 How to answer naturally
You: "I strictly use the Trigger Handler pattern. You'll never see me writing logic in the `.trigger` file itself." "I'd define the trigger, set the events like `before insert` or `after update`, and immediately pass `Trigger.new` or `Trigger.oldMap` to a helper class. It makes the code way easier to read, test, and maintain."
Code: Trigger Pattern
trigger AccountTrigger on Account (before insert, after update) {
    if(Trigger.isBefore && Trigger.isInsert) {
        AccountTriggerHandler.onBeforeInsert(Trigger.new);
    }
}

14. What is a Decorator and what is it used for in LWC?

🎤 How to answer naturally
You: "We basically use three main decorators in LWC to change how properties behave." "If I need to pass data from a parent component into a child, I use `@api` to make the child's property public. If I need to pull data from Salesforce automatically without imperative Apex, I use `@wire`. And I use `@track` if I have an array or object and I need the UI to update when data *inside* that object changes."

17. How do you communicate from a Parent component down to a Grandchild component?

🎤 How to answer naturally
You: "I try to avoid prop drilling. Passing an `@api` property from parent, to child, to grandchild gets messy fast." "Instead, I rely on Lightning Message Service (LMS). I just create a Message Channel. The parent component publishes the data, and the grandchild component subscribes to it. It totally bypasses the middle component and keeps the architecture decoupled."

19. How to use SOSL?

🎤 How to answer naturally
You: "I use SOSL specifically when I need to do a text search across multiple, unrelated objects. You start it with the `FIND` keyword." "So, `FIND 'john@gmail.com' IN EMAIL FIELDS RETURNING Account, Contact`. It gives you back a list of lists, which you can then parse out."

20. What is a Platform Event? Have you built anything using it?

🎤 How to answer naturally

Instead of my Apex trigger waiting around for an external system to respond via a slow HTTP callout, the trigger just published a Platform Event. 23. JS: How does async work? What are the ways we can configure it?

🎤 How to answer naturally
You: "Since JS doesn't block the thread, async operations like Apex callouts run in the background and return a Promise." "You can handle that promise the old way, using `.then()` and `.catch()`, but I prefer the modern way: `async/await`. You just tag the function as async, and use the await keyword before the Apex call. It pauses the code right there until the server responds, which makes the code way easier to read."
