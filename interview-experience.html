Write a basic trigger on the Account object.

🎤 How to answer naturally (5 YOE)
You (Architect Mode): "In an enterprise environment, I strictly follow a Trigger Handler Pattern. I never write logic inside the `.trigger` file. I use one trigger per object that delegates immediately to a handler class." "For a robust setup, I ensure the handler utilizes a static Set<Id> for recursion control to prevent the trigger from firing twice during complex workflows. I also usually implement a custom hierarchy setting (like an 'Automation Bypass' toggle) so we can turn off the trigger dynamically during data migrations without deploying code."
Code: Trigger Handler with Bypass
// The Thin Trigger
trigger AccountTrigger on Account (before insert, after update) {
    // Check Custom Setting to see if triggers are disabled for this user/profile
    if(TriggerSettings__c.getInstance().Disable_Account_Triggers__c) return;
    
    if(Trigger.isBefore && Trigger.isInsert) {
        AccountTriggerHandler.onBeforeInsert(Trigger.new);
    }
}

Tech Panel

An Account has multiple Contacts. A user shouldn't be able to delete a Contact if it is the "last associated" one. How do you solve this?

🎤 How to answer naturally
You: "I always clarify 'last associated' first." "If they mean the Final Remaining Contact (the account cannot be left empty), I use an Aggregate Query in a before delete trigger. I count the total contacts, subtract the ones being deleted, and if it drops below 1, I use addError(). Doing this math makes it perfectly safe for Mass-Deletes, whereas a Flow custom error might fail in bulk contexts." "If they mean the Chronologically Newest Linked Contact, standard CreatedDate won't work because a contact could be created Monday but linked Friday. I'd create a custom Account_Linked_Date__c stamped via Flow, and in my trigger, use ORDER BY Account_Linked_Date__c DESC LIMIT 1 to find and protect that specific record."

Tech Panel

When a Lead is converted, automatically create a Task for the Account owner. Flow or Apex?

🎤 How to answer naturally
You: "Flow is the best practice here. I'd use an After-Save Record-Triggered Flow on Lead. The condition is IsConverted = True (only when record is updated to meet the condition). I map the Task's `WhatId` to `ConvertedAccountId`." "If I absolutely had to use Apex, I'd loop `Trigger.new`, check the `IsConverted` flag change, collect the Account IDs, query the Account owners, and bulk insert the Tasks. But for simple record creation like this, Flow is much more maintainable."

Tech Panel

How do you handle errors in Flow?

🎤 How to answer naturally (5 YOE)
You: "A lot of people just drag a Fault Path to a screen element, but in an enterprise org, that's not enough. I build a standard Error Handler Subflow." "Whenever a DML element fails (like an 'Update Records'), I drag the Fault Path to this Subflow and pass in the $Flow.FaultMessage. The Subflow gracefully handles inserting a record into a custom Error_Log__c object so developers can debug it later, and it can optionally send an email alert to the admin team. It keeps all my main flows clean and standardizes how we track failures."

Tech Panel

Explain JSON structure and how you handle it in Salesforce.

🎤 How to answer naturally (5 YOE)
You: "At a base level, JSON is a lightweight format using key-value pairs. But in Apex integrations, how you handle it is what matters." "If I have a complex incoming JSON payload from an external API, I never parse it manually. I create a strongly-typed Wrapper Class in Apex, and use JSON.deserialize(jsonString, MyWrapper.class). It instantly maps the JSON to my Apex object." "If the JSON payload has dynamic keys that change constantly, I use JSON.deserializeUntyped(), which casts it to a Map<String, Object> so I can traverse it dynamically."

Tech Panel

Build an LWC search button for multiple objects via Email. Explain Async JS and SOSL.

🎤 How to answer naturally (5 YOE)
You: "I’d build a custom LWC exposed as a Quick Action. Since they are typing, I'd implement a debounce function in JavaScript—so we wait 300ms after they stop typing before hitting the server. It prevents burning through API limits." "When it fires, I use async/await to imperatively call an Apex method. Inside Apex, I use a SOSL (`FIND`) query. We have to use SOSL because we are searching a text string across multiple unrelated objects (Accounts and Contacts) simultaneously, which SOQL cannot do efficiently in one transaction." "I wouldn't use @wire(cacheable=true) if the user expects to edit those records immediately, because cacheable methods restrict DML."

Tech Panel

How do you communicate from a Parent component down to a Grandchild component?

🎤 How to answer naturally
You: "I avoid 'prop drilling' (passing `@api` properties through the middle child). It creates tight coupling." "Instead, I use Lightning Message Service (LMS). The Parent publishes a payload to a Message Channel, and the Grandchild subscribes to it directly. It’s the modern standard, replacing the old pubsub model, and it even allows communication across Aura, Visualforce, and LWC on the same page."
👤
Tech Panel

What is OmniStudio? Difference between PE and CDC?

🎤 How to answer naturally
You: "OmniStudio is a declarative toolset heavily used in Salesforce Industry clouds (like Health Cloud or Vlocity). We use it to build complex, pixel-perfect UIs via FlexCards and guided processes via OmniScripts without writing custom LWC/Apex." "For integration: CDC (Change Data Capture) is purely for automated data-syncing—turn it on, and it auto-publishes JSON when a record changes. Platform Events are for custom business logic—you define the schema and use Apex/Flow to explicitly publish them to external systems."
