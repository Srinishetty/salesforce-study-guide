HTTP Callouts — Calling External APIs from Apex

Salesforce can call any REST/SOAP endpoint. Always use Named Credentials for security. Remember: no callouts in synchronous triggers.

Basic REST Callout

public class ExternalApiService {
    
    // ✅ Using Named Credential (BEST PRACTICE)
    public static String getExternalData(String endpoint) {
        HttpRequest req = new HttpRequest();
        req.setEndpoint('callout:My_Named_Credential/' + endpoint);
        req.setMethod('GET');
        req.setHeader('Content-Type', 'application/json');
        req.setTimeout(30000); // 30 second timeout
        
        Http http = new Http();
        HttpResponse res = http.send(req);
        
        if (res.getStatusCode() == 200) {
            return res.getBody();
        } else {
            throw new CalloutException(
                'API Error: ' + res.getStatusCode() 
                + ' ' + res.getBody()
            );
        }
    }
    
    // POST with JSON body
    public static String createExternalRecord(Map<String,Object> payload) {
        HttpRequest req = new HttpRequest();
        req.setEndpoint('callout:My_Named_Credential/api/records');
        req.setMethod('POST');
        req.setHeader('Content-Type', 'application/json');
        req.setBody(JSON.serialize(payload));
        
        Http http = new Http();
        HttpResponse res = http.send(req);
        
        if (res.getStatusCode() == 201) {
            Map<String,Object> result = (Map<String,Object>) 
                JSON.deserializeUntyped(res.getBody());
            return (String) result.get('id');
        }
        throw new CalloutException('Create failed: ' + res.getBody());
    }
}

Callout Rules & Limits

RuleDetail
Max callouts100 per transaction
Max timeout120 seconds (sync), 120s (async)
Max response size12 MB (heap limit applies)
From triggers?❌ No — use @future(callout=true)
From batch?✅ Yes — implement Database.AllowsCallouts
After DML?❌ No — callout before DML, or use @future

Key rule: You cannot mix DML and callouts in the same transaction. Do callout first, then DML. Or use @future/@Queueable to separate them.

Custom REST & SOAP APIs — Exposing Salesforce

Create custom API endpoints in Apex that external systems can call. REST is simpler; SOAP generates a WSDL for enterprise clients.

Custom REST API

@RestResource(urlMapping='/accounts/*')
global with sharing class AccountRestService {
    
    @HttpGet
    global static Account getAccount() {
        RestRequest req = RestContext.request;
        // URL: /services/apexrest/accounts/001XX000003ABC
        String accId = req.requestURI.substringAfterLast('/');
        return [SELECT Id, Name, Industry, Phone
                FROM Account WHERE Id = :accId];
    }
    
    @HttpPost
    global static Id createAccount(String name, String industry) {
        Account acc = new Account(
            Name = name, 
            Industry = industry
        );
        insert acc;
        return acc.Id;
    }
    
    @HttpPut
    global static Account updateAccount(Id id, String name) {
        Account acc = [SELECT Id FROM Account WHERE Id = :id];
        acc.Name = name;
        update acc;
        return acc;
    }
    
    @HttpDelete
    global static void deleteAccount() {
        RestRequest req = RestContext.request;
        String accId = req.requestURI.substringAfterLast('/');
        delete [SELECT Id FROM Account WHERE Id = :accId];
    }
}

// External call:
// GET https://myDomain.my.salesforce.com/services/apexrest/accounts/001XX
// Headers: Authorization: Bearer <access_token>

Platform Events — Event-Driven Architecture

Platform Events enable loosely-coupled, real-time communication between Salesforce and external systems. Publish/Subscribe pattern — producers don't know consumers.

Define & Publish Events

// 1. Define: Setup → Platform Events → New
// Name: Order_Status__e (__e = event suffix)
// Fields: Order_Id__c (Text), Status__c (Text), 
//         Amount__c (Number)

// 2. Publish from Apex:
Order_Status__e event = new Order_Status__e(
    Order_Id__c = order.Id,
    Status__c = 'Shipped',
    Amount__c = order.Total_Amount__c
);
Database.SaveResult sr = EventBus.publish(event);

if (sr.isSuccess()) {
    System.debug('Event published: ' + sr.getId());
} else {
    System.debug('Error: ' + sr.getErrors()[0].getMessage());
}

// Publish multiple events (bulkified):
List<Order_Status__e> events = new List<Order_Status__e>();
for (Order ord : orders) {
    events.add(new Order_Status__e(
        Order_Id__c = ord.Id,
        Status__c = ord.Status
    ));
}
List<Database.SaveResult> results = EventBus.publish(events);

// Publish from Flow:
// Create Records element → Object: Order_Status__e

Subscribe to Events

// Subscribe in Apex Trigger:
trigger OrderStatusTrigger on Order_Status__e (after insert) {
    List<Task> tasks = new List<Task>();
    for (Order_Status__e event : Trigger.new) {
        if (event.Status__c == 'Shipped') {
            tasks.add(new Task(
                Subject = 'Follow up: Order shipped',
                WhatId = event.Order_Id__c,
                ActivityDate = Date.today().addDays(3)
            ));
        }
    }
    insert tasks;
}

// Subscribe in LWC (EMP API):
import { subscribe } from 'lightning/empApi';
subscribe('/event/Order_Status__e', -1, (message) => {
    const payload = message.data.payload;
    this.updateOrderStatus(payload.Order_Id__c, payload.Status__c);
});

Integration Best Practices & Error Handling

Retry Pattern with Error Logging

public class IntegrationService {
    private static final Integer MAX_RETRIES = 3;
    
    public static HttpResponse callWithRetry(HttpRequest req) {
        HttpResponse res;
        Integer retryCount = 0;
        
        while (retryCount < MAX_RETRIES) {
            try {
                Http h = new Http();
                res = h.send(req);
                
                if (res.getStatusCode() == 200 || 
                    res.getStatusCode() == 201) {
                    return res; // Success!
                }
                
                // 5xx = server error, retry
                if (res.getStatusCode() >= 500) {
                    retryCount++;
                    continue;
                }
                
                // 4xx = client error, don't retry
                break;
                
            } catch (CalloutException e) {
                retryCount++;
                if (retryCount >= MAX_RETRIES) {
                    logError('Callout failed after retries', e);
                    throw e;
                }
            }
        }
        
        logError('API returned: ' + res.getStatusCode(), null);
        return res;
    }
    
    private static void logError(String msg, Exception e) {
        Integration_Log__c log = new Integration_Log__c(
            Message__c = msg,
            Stack_Trace__c = e != null ? e.getStackTraceString() : '',
            Timestamp__c = Datetime.now()
        );
        insert log;
    }
}

Integration Checklist

  • ✅ Named Credentials — never hardcode secrets
  • ✅ Error handling — log failures, retry transients
  • ✅ Timeout handling — set appropriate timeouts
  • ✅ Bulkify — batch records in callout body
  • ✅ Idempotency — safe to retry (use external IDs)
  • ✅ Rate limiting — respect external API limits
  • ✅ Mock tests — 100% testable with HttpCalloutMock
  • ✅ Monitoring — custom object for integration logs
  • ✅ Circuit breaker — stop calling if endpoint is down

Pub/Sub API — gRPC-Based Event Streaming (2023+)

The modern replacement for CometD (EMP Connector) for subscribing to and publishing platform events at scale.

What Is Pub/Sub API?

  • Protocol: gRPC (binary, HTTP/2) — faster and more efficient than CometD (JSON/long-polling)
  • Replaces: EMP Connector / CometD for external subscribers
  • Supports: Platform Events, Change Data Capture (CDC), Custom Channels
  • Features: Bidirectional streaming, flow control, custom replayId management
  • Auth: OAuth 2.0 (same as REST/SOAP), uses session token or JWT
  • Endpoint: api.pubsub.salesforce.com:7443
  • Languages: Java, Python, Go, Node.js, .NET (any gRPC-supported language)
CometD vs Pub/Sub API Comparison:
FeatureCometD (Legacy)Pub/Sub API
ProtocolHTTP/1.1 long-pollinggRPC (HTTP/2 streams)
PayloadJSONAvro (binary, compact)
DirectionSubscribe onlySubscribe + Publish
Flow controlNone (flood risk)Client-managed batches
Performance~2K events/sec~10K+ events/sec
Replay24h window only72h + custom replayId

from pubsub_api_pb2_grpc import PubSubStub
from pubsub_api_pb2 import (
    FetchRequest, TopicInfo, ReplayPreset

# Connect via gRPC

    'api.pubsub.salesforce.com:7443',
    grpc.ssl_channel_credentials()
)
stub = PubSubStub(channel)

# Subscribe to platform event
def fetch_req_stream():
    while True:

            topic_name='/event/Order_Event__e',
            replay_preset=ReplayPreset.LATEST,
            num_requested=10  # Flow control!
        )

for event_batch in stub.Subscribe(
