Visualforce Fundamentals

Visualforce is a tag-based markup language similar to HTML/JSP. Pages run on the Salesforce server and render HTML to the browser. Still relevant for PDF generation, email templates, and legacy maintenance.

Your First Visualforce Page

<apex:page standardController="Account">
    <apex:pageBlock title="Account Details">
        <apex:pageBlockSection columns="2">
            <apex:outputField value="{!Account.Name}"/>
            <apex:outputField value="{!Account.Industry}"/>
            <apex:outputField value="{!Account.Phone}"/>
            <apex:outputField value="{!Account.AnnualRevenue}"/>
        </apex:pageBlockSection>
    </apex:pageBlock>

    <!-- Related Contacts -->
    <apex:pageBlock title="Contacts">
        <apex:pageBlockTable value="{!Account.Contacts}" var="c">
            <apex:column value="{!c.FirstName}"/>
            <apex:column value="{!c.LastName}"/>
            <apex:column value="{!c.Email}"/>
        </apex:pageBlockTable>
    </apex:pageBlock>
</apex:page>

Access at: /apex/PageName?id=001XX000003ABC

Architecture

// How VF works:
// 1. Browser requests /apex/MyPage?id=001XX
// 2. Server finds the VF page markup
// 3. Server calls the Controller to get data
// 4. Server renders HTML (server-side!)
// 5. HTML sent to browser (no client JS needed)

// Key difference from LWC:
// VF = Server-rendered (like PHP/JSP)
// LWC = Client-rendered (like React/Angular)

// Expression syntax: {!expression}
{!Account.Name}              // Field value
{!Account.Contacts.size}     // Collection size
{!YEAR(TODAY())}             // Formula function
{!IF(isAdmin, 'Yes', 'No')} // Conditional
{!URLFOR($Resource.MyZip, 'img/logo.png')} // Static resource

Visualforce Controllers

Controllers provide data and handle actions. Choose between Standard, Custom, and Extensions based on your needs.

Standard Controller (Zero Code)

<!-- Uses built-in Account controller -->
<apex:page standardController="Account">
    <!-- Automatically gets record from URL ?id=xxx -->
    <p>{!Account.Name}</p>
    
    <!-- Built-in actions: -->
    <apex:form>
        <apex:commandButton action="{!save}" value="Save"/>
        <apex:commandButton action="{!edit}" value="Edit"/>
        <apex:commandButton action="{!delete}" value="Delete"/>
        <apex:commandButton action="{!cancel}" value="Cancel"/>
    </apex:form>
</apex:page>

<!-- Standard List Controller (for list views) -->
<apex:page standardController="Account" recordSetVar="accounts">
    <apex:pageBlockTable value="{!accounts}" var="a">
        <apex:column value="{!a.Name}"/>
    </apex:pageBlockTable>
    <!-- Built-in pagination: -->
    <apex:commandLink action="{!next}" value="Next"/>
    <apex:commandLink action="{!previous}" value="Prev"/>
</apex:page>

Custom Controller (Full Control)

<!-- Page references the controller -->
<apex:page controller="AccountSearchController">
    <apex:form>
        <apex:inputText value="{!searchTerm}"/>
        <apex:commandButton value="Search" 
                           action="{!doSearch}"
                           reRender="results"/>
    </apex:form>
    <apex:outputPanel id="results">
        <apex:pageBlockTable value="{!results}" var="a">
            <apex:column value="{!a.Name}"/>
        </apex:pageBlockTable>
    </apex:outputPanel>
</apex:page>
// Apex Controller class
public with sharing class AccountSearchController {
    public String searchTerm { get; set; }
    public List<Account> results { get; set; }
    
    public AccountSearchController() {
        results = new List<Account>();
    }
    
    public void doSearch() {
        String term = '%' + String.escapeSingleQuotes(searchTerm) + '%';
        results = [SELECT Name, Industry, Phone
                   FROM Account 
                   WHERE Name LIKE :term
                   LIMIT 50];
    }
    
    // Page navigation
    public PageReference goToRecord() {
        return new PageReference('/' + selectedId);
    }
}

Controller Extensions & Advanced Patterns

Controller Extension (Best of Both)

<!-- Gets standard controller + custom logic -->
<apex:page standardController="Account" 
           extensions="AccountExtension">
    <p>{!Account.Name}</p>
    <p>Custom Data: {!relatedOpportunityCount}</p>
    <apex:commandButton value="Custom Action" 
                       action="{!doCustomAction}"/>
</apex:page>
// Extension class — augments standard controller
public with sharing class AccountExtension {
    private final Account acct;
    
    // MUST have this constructor signature
    public AccountExtension(ApexPages.StandardController std) {
        this.acct = (Account) std.getRecord();
    }
    
    public Integer getRelatedOpportunityCount() {
        return [SELECT COUNT() FROM Opportunity 
                WHERE AccountId = :acct.Id];
    }
    
    public PageReference doCustomAction() {
        // Custom logic here
        acct.Status__c = 'Reviewed';
        update acct;
        ApexPages.addMessage(new ApexPages.Message(
            ApexPages.Severity.CONFIRM, 'Account reviewed!'));
        return null; // Stay on same page
    }
}

PDF Generation (VF's Killer Feature)

<!-- Render entire page as PDF -->
<apex:page standardController="Account" 
           renderAs="pdf" 
           applyBodyTag="false">
    <head>
        <style>
            body { font-family: Arial, sans-serif; }
            .header { background: #003366; color: white; 
                      padding: 20px; }
            table { width: 100%; border-collapse: collapse; }
            td { border: 1px solid #ddd; padding: 8px; }
        </style>
    </head>
    <body>
        <div class="header">
            <h1>Invoice: {!Account.Name}</h1>
            <p>Date: {!TODAY()}</p>
        </div>
        <table>
            <apex:repeat value="{!Account.Opportunities}" var="opp">
                <tr>
                    <td>{!opp.Name}</td>
                    <td>{!opp.Amount}</td>
                </tr>
            </apex:repeat>
        </table>
    </body>
</apex:page>

VF Security Essentials

  • CRUD/FLS — Standard controller respects automatically; custom controller must check manually
  • XSS Prevention — Use {!HTMLENCODE(value)} or {!JSENCODE(value)} for user input
  • CSRF — Built-in ViewState token protects forms automatically
  • Sharing — Custom controllers should use with sharing
  • Output Encoding — <apex:outputText escape="true"/> (default)

VF Limitations in Lightning

  • iFrame — VF pages render inside an iframe (limited communication)
  • No $A — Aura framework not available in VF
  • JavaScript remoting — Still works but prefer LWC
  • ViewState limit — 170KB (watch complex pages)
  • No shadow DOM — CSS leaks possible
  • Mobile — Limited mobile support vs LWC
